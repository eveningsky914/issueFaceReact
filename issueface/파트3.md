# 결과 화면·시각화·캐릭터 표현 파트 설명

이 문서는 IssueFace 프로젝트에서 분석 결과를 화면에 보여주는 파트를 설명합니다.

## 서비스 전체 흐름
Home에서 국가/주제 선택
→ /analysis URL로 이동
→ useNewsAnalysis가 /api/analyze 호출
→ server/index.js가 Currents API + Google NL API로 분석
→ analysisResponseAdapter가 UI용 데이터로 변환
→ ResultHero / CountryCard / CompareStats가 화면에 표시



## 1. 사람이 읽는 설명

### 큰 틀

이 파트는 백엔드와 분석 hook이 만들어준 데이터를 사용자가 이해하기 쉽게 시각화합니다.

사용자는 결과 화면에서 다음 정보를 봅니다.

1. 상단 Hero 브리핑
2. 국가 간 감정 차이
3. 국가 간 캐릭터 상호작용 이미지
4. 국가별 분석 카드
5. tone 숫자와 tone bar
6. 핵심 보도 표현
7. 대표 뉴스 TOP 3
8. 상세 비교 지표
9. 분석 신뢰도
10. 종합 지표 레이더 차트

즉, 이 파트는 뉴스를 직접 가져오거나 Google Natural Language API를 호출하지 않습니다. 이미 만들어진 분석 데이터를 화면에 보기 좋게 배치하고, tone 값에 따라 색상/캐릭터/차트를 다르게 보여주는 역할을 합니다.

### 주요 흐름

```txt
Analysis.jsx
  -> ResultHero
  -> CountryCard 2개
  -> CompareStats
      -> ToneDistributionCompare
      -> AnalysisReliability
      -> OverviewRadarCompare
```

### 이 파트가 의존하는 주요 데이터

국가별 결과 카드는 대체로 아래 데이터를 사용합니다.

```js
data.hasData
data.tone
data.articleCount
data.allArticles
data.topNews
data.keyPhrases
data.keywords
data.toneDistribution
```

각 의미는 다음과 같습니다.

- `hasData`: 분석 가능한 뉴스가 있는지 여부
- `tone`: 국가별 평균 감정 점수
- `articleCount`: 분석에 사용된 기사 수
- `allArticles`: 분석된 기사 전체 목록
- `topNews`: 대표 뉴스 TOP 3
- `keyPhrases`: 핵심 보도 표현
- `keywords`: keyPhrases가 없을 때 쓰는 fallback 핵심어
- `toneDistribution`: 긍정/중립/부정 비율 데이터

### 발표할 때 강조할 점

- 이 파트는 분석 결과를 사용자가 이해하기 쉽게 시각화합니다.
- tone 값에 따라 색상, 캐릭터 이미지, 상호작용 이미지가 달라집니다.
- 상호작용 이미지는 neutral이 포함되지 않을 때만 표시됩니다.
- 상세 비교 지표는 tone 분포, 신뢰도, 종합 지표로 나뉩니다.
- 분석 신뢰도는 기사 수와 언론자유지수를 함께 사용합니다.

## 2. AI에게 전달할 프롬프트

아래 내용을 다른 AI에게 붙여넣으면 이 파트를 빠르게 이해시킬 수 있습니다.

```txt
나는 IssueFace React 프로젝트의 결과 화면·시각화·캐릭터 표현 파트를 맡고 있다.

이 파트는 이미 생성된 분석 데이터를 사용자에게 보기 좋게 보여주는 역할을 한다.
뉴스 API 호출이나 Google Natural Language API 분석은 직접 하지 않는다.

핵심 파일은 다음과 같다.

- src/pages/Analysis.jsx
  결과 화면 컨테이너다.
  URL 파라미터를 읽고 분석 hook을 실행하며, ResultHero, CountryCard 2개, CompareStats를 배치한다.

- src/components/result/ResultHero.jsx
  상단 Hero 브리핑 UI다.
  주제, 국가 vs 국가, 분석 시간, 감정 차이 카드, 국가 간 상호작용 이미지, 분석 요약을 보여준다.

- src/utils/analysisHero.js
  ResultHero에 필요한 heroData를 만든다.
  tone 차이, interactionType, normalizedPairKey, reactionSummary, 날짜 포맷 등을 계산한다.

- src/utils/interactionImages.js
  국가쌍과 interactionType에 맞는 상호작용 이미지를 반환한다.
  neutral이 포함되면 interactionType이 null이 되어 이미지가 표시되지 않는다.

- src/components/CountryCard.jsx
  국가별 분석 결과 카드다.
  국기, 국가명, 현재 감정, 큰 tone 숫자, 캐릭터 이미지, tone bar, 핵심 보도 표현, 대표 뉴스 TOP 3를 표시한다.

- src/components/ToneGauge.jsx
  -5~5 tone 범위를 bar로 시각화하고 marker 위치를 계산한다.

- src/utils/characterImages.js, src/utils/toneMood.js
  국가 코드와 tone 상태에 따라 캐릭터 이미지를 선택한다.
  tone 기준은 positive >= 0.3, negative <= -0.3, 그 사이는 neutral이다.

- src/components/CompareStats.jsx
  상세 비교 지표 컨테이너다.
  ToneDistributionCompare, AnalysisReliability, OverviewRadarCompare를 배치한다.

- src/components/ToneDistributionCompare.jsx
  양국의 긍정/중립/부정 비율을 누적 막대로 비교한다.

- src/components/AnalysisReliability.jsx
  기사 수와 언론자유지수를 바탕으로 분석 신뢰도를 계산해 보여준다.

- src/components/OverviewRadarCompare.jsx
  언론자유, 기사 수, 긍정 비율, 부정 비율을 레이더 차트로 비교한다.

- src/utils/toneDistribution.js
  allArticles의 toneScore를 기준으로 긍정/중립/부정 비율을 계산한다.

- src/utils/pressFreedom.js
  국가별 언론자유지수 데이터를 제공한다.

이 파트의 핵심은 API 결과를 직접 만드는 것이 아니라, data.tone, data.keyPhrases, data.topNews, data.allArticles, data.toneDistribution 같은 데이터를 받아 사용자에게 시각적으로 보여주는 것이다.
```
