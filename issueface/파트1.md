# 입력·탐색·페이지 흐름 파트 설명

이 문서는 IssueFace 프로젝트에서 사용자가 분석 조건을 만들고, 페이지를 이동하고, 히스토리를 관리하는 파트를 설명합니다.


## 서비스 전체 흐름
Home에서 국가/주제 선택
→ /analysis URL로 이동
→ useNewsAnalysis가 /api/analyze 호출
→ server/index.js가 Currents API + Google NL API로 분석
→ analysisResponseAdapter가 UI용 데이터로 변환
→ ResultHero / CountryCard / CompareStats가 화면에 표시





## 1. 사람이 읽는 설명

### 큰 틀

이 파트는 사용자가 서비스를 처음 만나는 부분입니다.

사용자는 홈 화면에서 다음 일을 합니다.

1. 지도 또는 3x3 국가 카드에서 비교할 국가 2개를 선택합니다.
2. 직접 주제를 입력하거나 프리셋 주제를 선택합니다.
3. `뉴스 분석 시작` 버튼을 눌러 분석 결과 페이지로 이동합니다.
4. 이후 히스토리 페이지에서 이전 분석 기록을 다시 보거나 삭제할 수 있습니다.

즉, 이 파트는 실제 뉴스 분석을 직접 수행하지는 않습니다. 대신 사용자가 분석에 필요한 조건을 고르고, 그 조건을 URL query 형태로 결과 페이지에 넘기는 역할을 합니다.

### 주요 흐름

```txt
Home 화면
  -> 국가 1 선택
  -> 국가 2 선택
  -> 주제 입력 또는 프리셋 선택
  -> 뉴스 분석 시작
  -> /analysis?country1=...&country2=...&topic=...&topicId=...
```

프리셋 버튼을 누르면 `topicId`가 함께 전달됩니다. 직접 입력한 주제는 `topicId` 없이 전달됩니다.

### 컴포넌트별 설명

| 파일 / 컴포넌트 | 역할 |
|---|---|
| `src/App.jsx` | 전체 라우팅을 담당합니다. 홈, 소개, 히스토리, 분석 결과, 국가 배경 페이지를 연결합니다. |
| `src/pages/Home.jsx` | 홈 화면의 컨테이너입니다. 국가 선택 상태, 주제 선택 상태, 분석 시작 로직을 관리합니다. |
| `src/components/WorldMap.jsx` | 세계 지도 UI입니다. 지원 국가 9개만 클릭 가능하며, 선택된 국가 색상을 지도에 반영합니다. |
| `src/components/home/CountrySelectionPanel.jsx` | 오른쪽 패널의 국가 선택 카드 영역입니다. 3x3 국가 카드와 ①/② 선택 표시를 담당합니다. |
| `src/components/home/TopicSelector.jsx` | 주제 선택 영역입니다. 직접 입력 input과 프리셋 버튼을 보여줍니다. |
| `src/components/home/AnalyzeButton.jsx` | 분석 시작 버튼입니다. 국가 2개와 주제가 있어야 활성화됩니다. |
| `src/data/homeOptions.js` | 홈 화면에서 쓰는 고정 데이터입니다. 프리셋 주제, 국가 카드 배치, 국가 한글명이 들어 있습니다. |
| `src/hooks/useCountries.jsx` | 로컬 국가 데이터를 UI에서 쓰기 좋은 형태로 제공합니다. |
| `src/data/countries.js` | 지원 국가 9개의 기본 정보입니다. 국가 코드, 이름, 국기 URL 등이 들어 있습니다. |
| `src/utils/supportedCountries.js` | 지도에서 선택 가능한 국가 코드 목록을 관리합니다. |
| `src/components/Header.jsx` | 모든 페이지 상단의 공통 헤더입니다. 소개, 히스토리, 사용법 메뉴를 보여줍니다. |
| `src/components/UsageModal.jsx` | 사용법 모달입니다. 서비스 사용 순서를 안내합니다. |
| `src/pages/Introduction.jsx` | 서비스 소개 페이지입니다. |
| `src/pages/CountryBackground.jsx` | 국가 배경 정보 페이지입니다. |
| `src/pages/History.jsx` | 분석 히스토리 페이지입니다. 기록 조회, 삭제, 다시 분석하기를 담당합니다. |
| `src/utils/historyStorage.js` | localStorage 기반 히스토리 저장 로직입니다. 저장, 삭제, 중복 제거를 처리합니다. |

### 발표할 때 강조할 점

- 이 파트는 “분석 조건을 만드는 사용자 흐름”을 담당합니다.
- 지도 선택과 카드 선택이 같은 상태를 공유합니다.
- 국가는 최대 2개까지만 선택할 수 있습니다.
- 프리셋 버튼을 누른 경우에만 `topicId`가 전달됩니다.
- 히스토리는 백엔드 없이 `localStorage`에 저장됩니다.





## 2. AI에게 전달할 프롬프트

아래 내용을 다른 AI에게 붙여넣으면 이 파트를 빠르게 이해시킬 수 있습니다.

```txt
나는 IssueFace React 프로젝트의 입력·탐색·페이지 흐름 파트를 맡고 있다.

이 파트는 사용자가 분석 조건을 만드는 화면과 페이지 이동 흐름을 담당한다.

핵심 파일은 다음과 같다.

- src/App.jsx
  전체 라우팅을 담당한다. /, /intro, /history, /analysis, /country/:cca2 경로를 연결한다.

- src/pages/Home.jsx
  홈 화면 컨테이너다. country1, country2, topicInput, selectedTopic, selectedTopicId 상태를 관리한다.
  지도와 오른쪽 패널 컴포넌트를 조립하고, 뉴스 분석 시작 시 /analysis로 이동한다.
  프리셋 버튼을 선택한 경우에만 topicId를 URL query에 포함한다.

- src/components/WorldMap.jsx
  react-simple-maps 기반 세계 지도다.
  지원 국가 9개만 클릭 가능하며, 선택된 국가1은 주황색, 국가2는 남색으로 표시한다.

- src/components/home/CountrySelectionPanel.jsx
  오른쪽 패널의 3x3 국가 선택 카드 UI다.
  국가 1은 ① 선택됨, 국가 2는 ② 선택됨으로 표시한다.

- src/components/home/TopicSelector.jsx
  주제 입력과 프리셋 버튼 UI다.
  직접 입력 시 selectedTopicId를 비우고, 프리셋 버튼 선택 시 topicId를 설정한다.

- src/components/home/AnalyzeButton.jsx
  뉴스 분석 시작 버튼이다.
  국가 2개와 주제가 없으면 disabled 상태가 된다.

- src/data/homeOptions.js
  홈 화면의 고정 옵션 데이터다.
  5개 프리셋 주제, 3x3 국가 배치, 국가 한글명을 담고 있다.

- src/hooks/useCountries.jsx, src/data/countries.js, src/utils/supportedCountries.js
  지원 국가 9개 데이터를 관리하고 지도 선택 가능 여부를 판단한다.

- src/pages/History.jsx, src/utils/historyStorage.js
  분석 히스토리를 localStorage에 저장하고 조회/삭제/다시 분석하기를 처리한다.

이 파트는 뉴스 수집이나 감정 분석을 직접 하지 않는다.
분석 조건을 만들고 URL query로 결과 페이지에 넘기는 것이 핵심 역할이다.
```
